# AfterMe-FE — Hệ thống đa ngôn ngữ (i18n)

## 1. Tổng quan

Hệ thống i18n được xây dựng theo đúng kiến trúc **feature-based**: mỗi feature tự quản lý bản dịch của mình, sau đó được gộp lại ở root `src/locales/` để `LanguageContext` tiêu thụ.

Mặc định ngôn ngữ hiển thị là **Tiếng Việt**. Lựa chọn được lưu vào `localStorage` và phục hồi khi reload trang.

---

## 2. Cấu trúc thư mục

```
src/
├── locales/
│   ├── en.ts          ← Combiner: gộp sharedEn + homeEn + ...
│   ├── vi.ts          ← Combiner: gộp sharedVi + homeVi + ...
│   └── index.ts       ← Re-export en, vi, Translations
│
├── app/
│   ├── LanguageContext.tsx  ← Context + Provider
│   └── useLanguage.ts       ← Hook: useLanguage()
│
├── shared/
│   └── locales/
│       ├── en.ts      ← Header, Footer (Tiếng Anh)
│       └── vi.ts      ← Header, Footer (Tiếng Việt)
│
└── features/
    └── home/
        └── locales/
            ├── en.ts  ← Hero, HowWork, MoreThan, TheRisk, WhoNeed, Pricing (EN)
            └── vi.ts  ← Hero, HowWork, MoreThan, TheRisk, WhoNeed, Pricing (VI)
```

---

## 3. Cách sử dụng trong component

Dùng hook `useLanguage()` để lấy object dịch `t`:

```tsx
import { useLanguage } from "@/app/useLanguage";

export const MyComponent = () => {
  const { t } = useLanguage();

  return <h1>{t.hero.heading1}</h1>;
};
```

Chuyển đổi ngôn ngữ:

```tsx
const { language, toggleLanguage, setLanguage } = useLanguage();

// Toggle qua lại
toggleLanguage();

// Đặt cụ thể
setLanguage("vi"); // hoặc "en"
```

## 4. Thêm bản dịch cho feature mới

Ví dụ thêm feature `dashboard`:

### Bước 1 — Tạo file locale trong feature

**`src/features/dashboard/locales/en.ts`**

```ts
export const dashboardEn = {
  dashboard: {
    title: "Dashboard",
    welcome: "Welcome back",
  },
};

export type DashboardTranslations = typeof dashboardEn;
```

**`src/features/dashboard/locales/vi.ts`**

```ts
import type { DashboardTranslations } from "./en";

export const dashboardVi: DashboardTranslations = {
  dashboard: {
    title: "Bảng điều khiển",
    welcome: "Chào mừng trở lại",
  },
};
```

### Bước 2 — Gộp vào root combiner

**`src/locales/en.ts`**

```ts
import { sharedEn } from "@/shared/locales/en";
import { homeEn } from "@/features/home/locales/en";
import { dashboardEn } from "@/features/dashboard/locales/en"; // thêm dòng này

export const en = {
  ...sharedEn,
  ...homeEn,
  ...dashboardEn, // thêm dòng này
};

export type Translations = typeof en;
```

**`src/locales/vi.ts`**

```ts
import { sharedVi } from "@/shared/locales/vi";
import { homeVi } from "@/features/home/locales/vi";
import { dashboardVi } from "@/features/dashboard/locales/vi"; // thêm dòng này

export const vi = {
  ...sharedVi,
  ...homeVi,
  ...dashboardVi, // thêm dòng này
};
```

### Bước 3 — Dùng trong component

```tsx
const { t } = useLanguage();
return <h1>{t.dashboard.title}</h1>;
```

## 5. Thêm ngôn ngữ mới (ví dụ: tiếng Nhật)

1. Tạo `ja.ts` trong từng `*/locales/` (shared, home, dashboard, ...)
2. Tạo `src/locales/ja.ts` gộp tất cả lại
3. Thêm `"ja"` vào kiểu `Language` trong `src/app/LanguageContext.tsx`
4. Thêm `ja` vào object `translations` trong `LanguageContext`
5. Cập nhật UI `LanguageToggle` để hiển thị tuỳ chọn thứ ba

---

## 6. Các file cốt lõi

| File                                       | Vai trò                                             |
| ------------------------------------------ | --------------------------------------------------- |
| `src/app/LanguageContext.tsx`              | Context + Provider, lưu lựa chọn vào `localStorage` |
| `src/app/useLanguage.ts`                   | Hook `useLanguage()` tiêu thụ context               |
| `src/shared/components/LanguageToggle.tsx` | Nút chuyển ngôn ngữ 🇻🇳 / 🇬🇧                         |
| `src/locales/en.ts` &amp; `vi.ts`          | Combiner gộp tất cả feature locales lại             |
| `src/shared/locales/`                      | Bản dịch dùng chung (header, footer)                |
| `src/features/*/locales/`                  | Bản dịch riêng của từng feature                     |

---

## 7. Quy tắc đặt tên key

- Dùng **camelCase** cho tên key: `ctaPrimary`, `howItWorks`
- Nhóm key theo **section** hoặc **component**: `hero.badge`, `footer.rights`
- Mỗi feature có **namespace riêng** để tránh xung đột: `dashboard.title`, `home.hero.badge`

---

> Khi cần thêm text mới, chỉ cần thêm key vào file locale của feature tương ứng. Không cần chạm vào `LanguageContext` hay `useLanguage`.
